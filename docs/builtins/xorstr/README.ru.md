# Шифрование строк на этапе компиляции (`xorstr`)

## Обзор

NeverC предоставляет двухуровневое шифрование строк на этапе компиляции для кода на C, предназначенное для сценариев безопасности, где открытые строки (имена API, пути реестра) не должны быть видны в скомпилированном бинарном файле.

- **Уровень 1 — Явный макрос**: `NC_XORSTR("string")` / `NEVERC_XORSTR("string")` для точного управления каждой строкой
- **Уровень 2 — Автоматический IR-проход**: `-fencrypt-call-strings` для автоматического шифрования всех строковых аргументов в вызовах функций

Оба уровня используют буферы на стеке (без выделения в куче), алгоритм дешифрования без инструкции XOR (антисигнатура) и очистку volatile `memset` перед возвратом из функции.

---

## Быстрый старт

```c
#include <neverc/xorstr.h>
FARPROC addr = GetProcAddress(hModule, NC_XORSTR("NtQuerySystemInformation"));
```

```bash
neverc -fencrypt-call-strings main.c -o main
```

---

## Антисигнатурное дешифрование

Операция дешифрования полностью избегает инструкции XOR, используя математически эквивалентное: `a + b − 2 × (a & b)`.

---

## Справочник флагов компилятора

| Флаг | Описание |
|------|----------|
| `-fencrypt-call-strings` | Включить автоматическое шифрование строк |
| `-fno-encrypt-call-strings` | Отключить автоматическое шифрование |
| `-fencrypt-call-strings-max-len=N` | Максимальная длина в байтах (по умолчанию: 1024) |
| `-fstring-encrypt-key=0xHEX` | Переопределить начальное значение ключа XOR |
